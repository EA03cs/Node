ليه محتاجين Database ومش File System؟

تخيل معايا إنك بتشتغل على نظام مدرسة وقررت تخزن البيانات في ملفات نصية عادية .txt أو .json...

📁 مشاكل الـ File System
  • Data Redundancy — اسم الطالب متكرر في ملف الطلاب وملف الدرجات وملف الحضور
  • Data Inconsistency — لو غيرت في ملف ونسيت التاني، البيانات بتتعارض
  • صعوبة الـ Query — عايز تجيب الطلاب فوق 18 وشاطرين؟ كود طويل جداً
  • مفيش Security — الملف موجود أو مش موجود، مفيش صلاحيات
  • Concurrency — لو اتنين شغلوا على نفس الملف، البيانات بتتلخبط
  • Backup & Recovery — لو الملف اتحذف خلاص راح
🗄️ مزايا الـ Database
  • Data Integrity — بيانات دايماً صح ومتناسقة
  • Security — "المحاسب يشوف المرتبات بس"
  • Concurrent Access — 100 مستخدم في نفس الوقت
  • Backup & Recovery — نظام متكامل للنسخ الاحتياطي
  • Query سهل — سطر SQL واحد يجيبلك أي حاجة
  • Reduced Redundancy — كل بيانة مرة واحدة بس
💡 الـ File System مناسب جداً لتطبيقات بسيطة زي قراءة ملف إعدادات. لكن لما البيانات تبقى كبيرة ومترابطة، الـ Database هو الحل الوحيد.

إيه هو الـ Database؟

الـ Database هو مجموعة منظمة من البيانات المترابطة مخزنة بطريقة تسمح بالوصول إليها وإدارتها وتحديثها بسهولة.

📦 الـ DB (قاعدة البيانات)
  • البيانات نفسها
  • الجداول والمعلومات
  • زي الكتب في المكتبة
⚙️ الـ DBMS (نظام الإدارة)
  • البرنامج اللي بيدير الـ DB
  • MySQL, Oracle, SQL Server
  • زي أمين المكتبة

مثال عملي — بيانات الطلاب في جدول:

رقم الطالبالاسمالسنالقسمالإيميل
001Mohamed Ali20Computer Sciencemo@email.com
002Ahmed Hassan22Information Systemsah@email.com
003Sara Omar19Computer Scienceso@email.com
004Ali Mahmoud21ITal@email.com

أنواع الـ Databases — SQL vs NoSQL
🗃️ SQL (Relational)
  • البيانات في جداول منظمة
  • الجداول مترابطة ببعض
  • MySQL, PostgreSQL, Oracle
  • SQL Server, SQLite
  • مناسبة لـ: بنك، مدرسة، شركة
📄 NoSQL (Non-Relational)
  • البيانات بأشكال مختلفة (JSON, Key-Value)
  • MongoDB — بيخزن JSON Documents
  • Redis — بيخزن Key-Value
  • Cassandra — Column-based
  • مناسبة لـ: سوشيال ميديا، Big Data

متى تختار إيه؟ جدول المقارنة:

الاستخدامSQLNoSQL
بيانات منظمة ومترابطة✓ ممتاز⚠ ممكن
سرعة القراءة والكتابة العالية⚠ متوسط✓ ممتاز
بيانات كبيرة جداً (Big Data)⚠ صعب✓ ممتاز
Transactions معقدة✓ ممتاز✗ صعب
سهولة التعلم✓ أسهل⚠ أصعب شوية
Schema ثابت✓ نعم⚠ مرن (Schema-less)

مزايا وعيوب الـ Database
✅ المزايا
  • Data Integrity — بيانات صح دايماً
  • Security — صلاحيات محددة لكل مستخدم
  • Concurrent Access — كتير مستخدمين مع بعض
  • Backup & Recovery — استرجاع البيانات
  • Data Sharing — كل الأقسام على نفس الـ DB
  • Reduced Redundancy — كل بيانة مرة واحدة
❌ العيوب
  • Cost — Oracle وSQL Server بيكلفوا فلوس
  • Complexity — محتاج DBA متخصص يديره
  • Overhead — أبطأ من File System في الحالات البسيطة
  • Single Point of Failure — لو الـ DB وقع، كل حاجة وقفت

الـ ER Model وأنواع الـ Keys

الـ ER Model هو طريقة تصميم بتبين الكيانات في النظام والعلاقات بينهم.

المكونات الأساسية
  • Entity — أي حاجة في الواقع: طالب، مادة، أستاذ
  • Attribute — صفة الـ Entity: اسم، سن، عنوان
  • Relationship — الرابط بين Entities: طالب "يسجل في" مادة
Strong vs Weak Entity
  • Strong Entity — مستقل وعنده PK خاص بيه
  • Weak Entity — يعتمد على Strong Entity، عنده Partial Key بس
  • مثال: DEPENDENT (تابع) يعتمد على EMPLOYEE

أنواع الـ Keys:

Primary Key (PK)
معرف فريد لكل صف — لازم يكون Unique ومش Null أبداً
مثال: Student_ID
Foreign Key (FK)
الـ PK من جدول تاني موجود في جدول عشان يربط الجدولين
مثال: Dept_ID في جدول EMPLOYEE
Composite Key
مفتاح مكون من أكتر من عمود مع بعض عشان يبقى Unique
مثال: (Student_ID + Course_ID)
Candidate Key
أي عمود ممكن يكون PK — بنختار واحد منهم يبقى الـ PK
مثال: الرقم القومي أو رقم الطالب
Super Key
أي مجموعة أعمدة تعرف الصف بشكل فريد حتى لو فيها زيادة
مثال: (ID + Name) — الـ Name زيادة
Partial Key
بس في Weak Entity — مش كافي لوحده يعرف الـ Entity
مثال: Dep_Name في DEPENDENT

رموز ومفاهيم الـ ERD

ملخص الرموز المستخدمة في رسم الـ ERD:

الشكلالاسمالوصفمثال
[ ] مستطيلStrong Entityكيان مستقل عنده PK خاصSTUDENT, EMPLOYEE
[[ ]] مزدوجWeak Entityيعتمد على كيان تانيDEPENDENT
( ) بيضاويAttributeصفة للـ EntityName, Age
(__) بخط تحتKey Attributeالـ Primary KeyStudent_ID
(( )) مزدوجMultivalued Attrممكن يكون ليه أكتر من قيمةPhone Numbers
(---) منقطDerived Attrمتحسب من حاجة تانيةAge من Birthdate
< > ماسةRelationshipعلاقة بين EntitiesWorksFor, Manages
<< >> مزدوجIdentifying Relعلاقة الـ Weak Entity بالـ OwnerDependentOf
خط واحدPartial Participationالمشاركة اختياريةEmployee → Manages
خط مزدوجTotal Participationالمشاركة إجباريةEmployee → WorksFor

أنواع العلاقات (Cardinality):

1 : 1
  • كل طرف مرتبط بطرف واحد بالكتير
  • مثال: موظف يدير قسم واحد
  • والقسم عنده مدير واحد
1 : N
  • طرف واحد يرتبط بكتير من الطرف التاني
  • مثال: قسم فيه كتير موظفين
  • كل موظف في قسم واحد بس
M : N
  • كل الطرفين يرتبطوا بكتير
  • مثال: طالب يسجل في مواد كتير
  • والمادة فيها طلاب كتير
💡 علاقة M:N دايماً بتنتج جدول وسطي (Junction Table) في مرحلة الـ Mapping!

الـ Mapping — تحويل ERD لجداول

الـ Mapping هو تحويل رسمة الـ ERD لجداول SQL فعلية. في 6 قواعد أساسية.

قاعدة ١Strong Entity → جدول
كل Entity بتبقى جدول. كل Attribute بيبقى عمود. الـ Key Attribute بيبقى Primary Key.
SQL
-- الـ ERD: EMPLOYEE (Ssn, Fname, Lname, Salary, Bdate)
--         Ssn هو الـ Key Attribute

CREATE TABLE EMPLOYEE (
    Ssn    CHAR(9)       PRIMARY KEY,
    Fname  VARCHAR(30)  NOT NULL,
    Lname  VARCHAR(30)  NOT NULL,
    Salary DECIMAL(10,2),
    Bdate  DATE
);
قاعدة ٢Weak Entity → جدول + FK من الـ Owner
بتعمل جدول وبتضيف الـ PK من الـ Owner كـ FK. الـ PK الجديد = (Partial Key + FK من الـ Owner).
SQL
-- الـ ERD: DEPENDENT (Dep_name, Sex, Bdate) - Weak Entity
-- تابع للـ EMPLOYEE عن طريق علاقة DEPENDENTS_OF

CREATE TABLE DEPENDENT (
    Essn      CHAR(9),          -- FK من EMPLOYEE
    Dep_name  VARCHAR(30),    -- Partial Key
    Sex       CHAR(1),
    Bdate     DATE,
    Relation  VARCHAR(20),
    PRIMARY KEY (Essn, Dep_name),   -- PK مركب
    FOREIGN KEY (Essn) REFERENCES EMPLOYEE(Ssn)
);
قاعدة ٣علاقة 1:1 → FK في أحد الجدولين
الـ PK من أي جدول يتحط كـ FK في الجدول التاني. الأفضل في الجدول اللي Total Participation.
SQL
-- علاقة MANAGES بين EMPLOYEE و DEPARTMENT (1:1)
-- كل قسم عنده مدير واحد (Total) → نحط الـ FK في DEPARTMENT

ALTER TABLE DEPARTMENT
ADD COLUMN Mgr_ssn       CHAR(9),      -- FK المدير
ADD COLUMN Mgr_start_date DATE,
ADD FOREIGN KEY (Mgr_ssn) REFERENCES EMPLOYEE(Ssn);
قاعدة ٤علاقة 1:N → FK في جدول طرف الـ N
الـ PK من طرف الـ 1 بيتحط كـ FK في جدول طرف الـ N.
SQL
-- علاقة WORKS_FOR: قسم واحد (1) فيه موظفين كتير (N)
-- نحط الـ FK (رقم القسم) في جدول EMPLOYEE جهة الـ N

ALTER TABLE EMPLOYEE
ADD COLUMN Dno INT,
ADD FOREIGN KEY (Dno) REFERENCES DEPARTMENT(Dnumber);
قاعدة ٥علاقة M:N → جدول جديد (Junction Table)
لازم تعمل جدول جديد. الـ PK بتاعه مركب من الـ PKs من الجدولين.
SQL
-- علاقة WORKS_ON: موظف (M) يشتغل على مشاريع كتير (N)
-- نعمل جدول جديد WORKS_ON

CREATE TABLE WORKS_ON (
    Essn  CHAR(9),    -- FK من EMPLOYEE
    Pno   INT,         -- FK من PROJECT
    Hours DECIMAL(5,1),
    PRIMARY KEY (Essn, Pno),    -- PK مركب
    FOREIGN KEY (Essn) REFERENCES EMPLOYEE(Ssn),
    FOREIGN KEY (Pno)  REFERENCES PROJECT(Pnumber)
);
قاعدة ٦Multivalued Attribute → جدول جديد
الـ Attribute اللي بيكون ليه أكتر من قيمة بيتحول لجدول جديد فيه الـ Attribute + الـ FK.
SQL
-- القسم ممكن يكون في أكتر من Location
-- Locations هو Multivalued Attribute في DEPARTMENT

CREATE TABLE DEPT_LOCATIONS (
    Dnumber   INT,          -- FK من DEPARTMENT
    Dlocation VARCHAR(50),
    PRIMARY KEY (Dnumber, Dlocation),
    FOREIGN KEY (Dnumber) REFERENCES DEPARTMENT(Dnumber)
);

الـ DDL — Data Definition Language

الـ DDL بيحدد هيكل الـ Database — مش بيشتغل على البيانات نفسها. الأوامر: CREATE, ALTER, DROP, TRUNCATE.

الأمرالوظيفة
CREATEإنشاء Database أو Table
ALTERتعديل هيكل جدول موجود (زيادة عمود، حذفه، تغيير نوعه)
DROPحذف جدول أو Database بالكامل (البيانات + الهيكل)
TRUNCATEمسح كل البيانات مع الإبقاء على هيكل الجدول
مثال ١ — CREATE DATABASE
SQL — CREATE
-- إنشاء قاعدة بيانات جديدة
CREATE DATABASE school;

-- تحديد إننا هنشتغل عليها
USE school;
مثال ٢ — CREATE TABLE بسيط
SQL — CREATE TABLE
CREATE TABLE students (
    id      INT           PRIMARY KEY,
    name    VARCHAR(50)   NOT NULL,
    age     INT,
    email   VARCHAR(100)
);
مثال ٣ — CREATE TABLE مع Foreign Key
SQL — FK Example
-- أولاً: الجدول الأساسي (Referenced Table)
CREATE TABLE departments (
    dept_id   INT          PRIMARY KEY,
    dept_name VARCHAR(50) NOT NULL
);

-- ثانياً: الجدول اللي بيشاور عليه (Referencing Table)
CREATE TABLE employees (
    emp_id   INT           PRIMARY KEY AUTO_INCREMENT,
    emp_name VARCHAR(50)  NOT NULL,
    salary   DECIMAL(10,2),
    dept_id  INT,
    CONSTRAINT FK_Dept FOREIGN KEY (dept_id)
              REFERENCES departments(dept_id)
);
⚠️ الـ VARCHAR(50) بيحجز المساحة اللي بتحتاجها بس، أما الـ CHAR(50) بيحجز 50 حرف ثابت حتى لو كتبت 5 حروف بس.
مثال ٤ — ALTER TABLE (إضافة عمود)
SQL — ALTER
-- إضافة عمود جديد
ALTER TABLE students ADD phone VARCHAR(15);

-- حذف عمود
ALTER TABLE students DROP COLUMN age;

-- تغيير حجم عمود
ALTER TABLE students MODIFY name VARCHAR(100);

-- إعادة تسمية عمود (MySQL 8+)
ALTER TABLE students RENAME COLUMN name TO full_name;
مثال ٥ — DROP vs TRUNCATE
SQL — DROP vs TRUNCATE
-- DROP: بيمسح الجدول بالكامل (البيانات + الهيكل)
-- بعدها مش هتلاقي الجدول خالص
DROP TABLE students;

-- TRUNCATE: بيمسح البيانات بس والجدول يفضل موجود
TRUNCATE TABLE students;

-- DROP DATABASE: حذف كل القاعدة
DROP DATABASE school;
١٠
الـ DML — Data Manipulation Language

الـ DML بيشتغل على البيانات الموجودة جوا الجداول. الأوامر: INSERT, SELECT, UPDATE, DELETE.

مثال ١ — INSERT (إضافة بيانات)
SQL — INSERT
-- إضافة صف كامل (لازم ترتيب القيم يطابق ترتيب الأعمدة)
INSERT INTO students VALUES (1, 'Mohamed Ali', 20, 'mo@email.com');

-- إضافة أعمدة محددة بس
INSERT INTO students (id, name, age)
VALUES (2, 'Ahmed Hassan', 22);

-- إضافة أكتر من صف دفعة واحدة
INSERT INTO students (id, name, age) VALUES
(3, 'Sara Omar',    19),
(4, 'Ali Mahmoud',  21),
(5, 'Nour Karim',   20);
مثال ٢ — SELECT الأساسي
SQL — SELECT
-- جيب كل البيانات
SELECT * FROM students;

-- جيب أعمدة محددة بس
SELECT name, age FROM students;

-- DISTINCT: شيل التكرار
SELECT DISTINCT age FROM students;

-- ALIAS: غير اسم العمود في العرض
SELECT name AS 'اسم الطالب', age AS 'العمر'
FROM students;
مثال ٣ — WHERE (الفلترة)
SQL — WHERE
-- طلاب سنهم أكبر من 20
SELECT * FROM students WHERE age > 20;

-- طالب معين برقمه
SELECT * FROM students WHERE id = 1;

-- BETWEEN: طلاب سنهم بين 18 و 22
SELECT * FROM students WHERE age BETWEEN 18 AND 22;

-- IN: طلاب رقمهم 1 أو 3 أو 5
SELECT * FROM students WHERE id IN (1, 3, 5);

-- AND: طلاب سنهم > 20 وكمان email موجود
SELECT * FROM students
WHERE age > 20 AND email IS NOT NULL;

-- OR: طلاب سنهم > 22 أو اسمهم يبدأ بـ A
SELECT * FROM students
WHERE age > 22 OR name LIKE 'A%';
مثال ٤ — LIKE (البحث بالنص)
SQL — LIKE
-- اسمه بيبدأ بـ M  (M%)
SELECT * FROM students WHERE name LIKE 'M%';

-- اسمه بينتهي بـ n  (%n)
SELECT * FROM students WHERE name LIKE '%n';

-- اسمه فيه ali في أي مكان  (%ali%)
SELECT * FROM students WHERE name LIKE '%ali%';

-- _ تعني حرف واحد بالظبط
SELECT * FROM students WHERE name LIKE 'A___';
-- هيجيب أسماء من 4 حروف تبدأ بـ A
مثال ٥ — ORDER BY (الترتيب)
SQL — ORDER BY
-- من الأصغر للأكبر (ASC — الافتراضي)
SELECT * FROM students ORDER BY age ASC;

-- من الأكبر للأصغر (DESC)
SELECT * FROM students ORDER BY age DESC;

-- ترتيب بأكتر من عمود
SELECT * FROM students
ORDER BY age DESC, name ASC;
مثال ٦ — Aggregate Functions + GROUP BY
SQL — GROUP BY
-- COUNT: عدد الطلاب الكلي
SELECT COUNT(*) AS total FROM students;

-- SUM: مجموع المرتبات
SELECT SUM(salary) FROM employees;

-- AVG: متوسط السن
SELECT AVG(age) AS avg_age FROM students;

-- MAX, MIN
SELECT MAX(salary), MIN(salary) FROM employees;

-- GROUP BY: عدد الطلاب في كل قسم
SELECT dept_id, COUNT(*) AS student_count
FROM students
GROUP BY dept_id;

-- HAVING: الأقسام اللي عندها أكتر من 5 طلاب
SELECT dept_id, COUNT(*) AS cnt
FROM students
GROUP BY dept_id
HAVING COUNT(*) > 5;
💡 الفرق بين WHERE وHAVING: الـ WHERE بيفلتر الصفوف قبل الـ GROUP BY، والـ HAVING بيفلتر النتائج بعد الـ GROUP BY.
مثال ٧ — JOIN
SQL — JOINs
-- INNER JOIN: جيب المشترك فقط (الطلاب اللي عندهم قسم)
SELECT s.name, s.age, d.dept_name
FROM students s
INNER JOIN departments d ON s.dept_id = d.dept_id;

-- LEFT JOIN: كل الطلاب حتى لو مش مسجلين في قسم
SELECT s.name, d.dept_name
FROM students s
LEFT JOIN departments d ON s.dept_id = d.dept_id;
-- الطالب اللي مش في قسم هيطلع dept_name = NULL

-- RIGHT JOIN: كل الأقسام حتى لو مفيش طلاب فيها
SELECT s.name, d.dept_name
FROM students s
RIGHT JOIN departments d ON s.dept_id = d.dept_id;

-- FULL JOIN: كل حاجة من الجدولين
SELECT s.name, d.dept_name
FROM students s
FULL JOIN departments d ON s.dept_id = d.dept_id;
مثال ٨ — UPDATE
SQL — UPDATE
-- تعديل سن طالب واحد
UPDATE students SET age = 21 WHERE id = 1;

-- تعديل أكتر من عمود في نفس الوقت
UPDATE students
SET   name  = 'Mohamed Ahmed',
      email = 'new@email.com'
WHERE id = 1;

-- زود راتب كل موظفين القسم 2 بـ 10%
UPDATE employees
SET   salary = salary * 1.10
WHERE dept_id = 2;
⚠️ دايماً اتأكد من الـ WHERE في الـ UPDATE — لو نسيته هيعدل كل الجدول!
مثال ٩ — DELETE
SQL — DELETE
-- حذف طالب واحد
DELETE FROM students WHERE id = 2;

-- حذف بشرط
DELETE FROM students WHERE age < 18;

-- حذف كل البيانات (أبطأ من TRUNCATE)
DELETE FROM students;
مثال ١٠ — مثال كامل من الصفر لآخر
SQL — Full Example
-- ══════════ 1. إنشاء قاعدة البيانات ══════════
CREATE DATABASE school;
USE school;

-- ══════════ 2. إنشاء الجداول ══════════
CREATE TABLE departments (
    dept_id   INT          PRIMARY KEY AUTO_INCREMENT,
    dept_name VARCHAR(50) NOT NULL
);

CREATE TABLE students (
    id      INT          PRIMARY KEY AUTO_INCREMENT,
    name    VARCHAR(50) NOT NULL,
    age     INT,
    email   VARCHAR(100),
    dept_id INT,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

-- ══════════ 3. إضافة بيانات ══════════
INSERT INTO departments (dept_name) VALUES
('Computer Science'), ('Information Systems'), ('IT');

INSERT INTO students (name, age, email, dept_id) VALUES
('Mohamed Ali',   20, 'mo@email.com', 1),
('Ahmed Hassan',  22, 'ah@email.com', 1),
('Sara Omar',     19, 'so@email.com', 2),
('Ali Mahmoud',   21, 'al@email.com', 3);

-- ══════════ 4. استعلامات ══════════
-- جيب كل الطلاب مع أسماء أقسامهم
SELECT s.name, s.age, d.dept_name
FROM students s
INNER JOIN departments d ON s.dept_id = d.dept_id

XAMPP و MySQL GUI

XAMPP هو برنامج بيثبت على جهازك مجموعة أدوات متكاملة دفعة واحدة.

مكونات XAMPP
  • Apache — Web Server بيشغل المواقع محلياً
  • MySQL — Database Server
  • PHP — لغة برمجة
  • phpMyAdmin — واجهة بصرية (GUI) من المتصفح
إيه ممكن تعمله في phpMyAdmin
  • تنشئ Database جديدة بضغطة زر
  • تنشئ Tables وتضيف Columns بـ GUI
  • تكتب SQL Queries في تبويب SQL
  • تستورد وتصدر بيانات

خطوات التشغيل:

١
نزل XAMPP من apachefriends.org واختار الإصدار المناسب لنظامك
٢
بعد التثبيت، افتح XAMPP Control Panel
٣
اضغط Start جنب Apache وجنب MySQL — اللمبة تبقى خضراء
٤
افتح المتصفح واكتب http://localhost/phpmyadmin
٥
من هنا تقدر تنشئ الـ DB، تعمل الجداول، وتكتب SQL مباشرة
